Team Reflection

Building the Paris Travel Planner from a blueprint into a fully conversational, multi-tool AI agent taught our team more about real-world engineering than any single concept from the course. What looked straightforward on paper — a LangChain agent with a few tools and a chat interface — turned into a ten-version iterative process where nearly every assumption we made in our Midterm plan was challenged: our chosen framework broke in production, our primary API hit its daily limit mid-demo, our tools crashed and corrupted memory in ways we had never anticipated, and the LLM itself kept formatting its own outputs incorrectly. Each failure forced us to go deeper — we did not just fix bugs, we rebuilt the architecture around LangGraph, engineered a four-model fallback chain, and refactored every tool to be defensively coded from the ground up. The result is something we are genuinely proud of: an agent that holds a real conversation, respects physical constraints a human planner would care about, and keeps working even when individual components fail. More than the technical output, this project taught us how to communicate under pressure, divide ownership of a complex system across three people, and make fast architectural decisions when the original plan stops working — which, in AI engineering, is most of the time.
